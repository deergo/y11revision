7. Algorithm design and problem-solving

  • Development cycle: analysis ‚Üí design ‚Üí implementation (coding) ‚Üí testing ‚Üí evaluation/maintenance.
  • Abstraction removes detail; decomposition/top‚Äëdown design breaks a problem into smaller sub‚Äëproblems; define clear inputs/outputs for each part.
  • Representation: flowcharts and pseudocode to describe algorithms before coding.
  • Standard algorithms:
    • Linear search (sequential): scan until found or end.
    • Bubble sort: repeatedly swap adjacent out‚Äëof‚Äëorder items until no swaps.
    • Totals/counting; finding max/min/average: loop through items maintaining running values.
  • Validation vs verification:
    • Validation checks data is reasonable (range, length, type, presence/required, format, check digit).
    • Verification checks data is entered/transferred correctly (double entry, visual check).
  • Test data types: normal, boundary/extreme, abnormal/invalid. Use trace tables to dry‚Äërun logic and locate errors.

8.2 Arrays

  • Use arrays to store multiple items of the same type under one name.
  • 1D arrays: index with a single subscript (first index might be 0 or 1).
  • 2D arrays: table/matrix; access with [row][column].
  • Use variables as indexes and nested loops to traverse 2D arrays.

9. Databases

  • Structure: tables consist of records (rows) with fields (columns). Common data types: text/alphanumeric, character, Boolean, integer, real, date/time.
  • SQL basics:
    • SELECT fields FROM table WHERE condition ORDER BY field;
    • Aggregate functions: COUNT(*), SUM(field); often used with GROUP BY.
  • Be able to predict the output of simple SQL queries.
# Example SQL
SELECT Name, Score
FROM   Results
WHERE  Score >= 50
ORDER BY Score DESC;

SELECT COUNT(*) AS Passed
FROM   Results
WHERE  Score >= 50;
