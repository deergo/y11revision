2. Data transmission

2.1 Types and methods of data transmission

  • Serial vs parallel:
    • Serial sends bits one after another on one pair/line: lower crosstalk, reliable over longer distances, cheaper cabling.
    • Parallel sends multiple bits at once on multiple lines: higher throughput over very short distances but suffers from skew/interference.
  • Direction: simplex (one-way), halfâ€‘duplex (both ways, not simultaneously), fullâ€‘duplex (both ways simultaneously).
  • USB: a serial bus with device detection and drivers.

    • Advantages: plugâ€‘andâ€‘play, standardised connectors/protocols, widely supported. Limits: cable length; older versions (e.g., USB 2.0) are halfâ€‘duplex; speed depends on version.
  • Packet switching: data split into packets (header + payload + trailer). Packets can take different routes; improves resilience and utilisation but can increase latency/jitter.

2.2 Methods of error detection

  • Parity bit (even/odd): detects singleâ€‘bit errors in a byte; cannot detect all multiâ€‘bit errors.
  • Parity block (2D parity): parity on rows and columns to locate a single erroneous bit.
  • Checksum: sender computes a value over a block; receiver recomputes and compares. Mismatch â†’ request resend.
  • Echo check: receiver returns data; sender compares with original.
  • Check digit: algorithmic extra digit for identifiers (e.g., ISBN, barcodes) to detect entry errors.
  • Error control (ARQ): acknowledgements + timeout. If ACK not received in time or NACK received â†’ retransmit.

2.3 Encryption

  • HTTPS uses TLS: combines certificates (public keys) with symmetric session keys for performance and security.
